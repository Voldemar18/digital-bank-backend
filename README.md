## 🏗️ Microservices Architecture

| Название папки | Объяснение |
|---------|-------------|
| **API Gateway** | Единая точка входа для всех внешних запросов. Проверяет JWT-токены и маршрутизирует трафик к нужным сервисам. Не имеет своей БД, работает как реактивный распределитель. |
| **User Service** | Отвечает за регистрацию, логин и верификацию (KYC). Хранит пользователей в user_db, пароли хэширует BCrypt, выдает JWT-токены. Меняет статус на VERIFIED по запросу. |
| **Account Service** | Управляет счетами и переводами. Верифицированным пользователям открывает кошельки в валютах, проводит P2P-переводы с Pessimistic Lock. После успешной операции отправляет событие в Kafka. |
| **Transaction Service** | Слушает Kafka и сохраняет историю всех операций в transaction_db. Выдает пользователям выписки через HTTP GET. Не принимает запросы на запись напрямую. |
| **Payment Gateway Mock** | Заглушка внешних платежных систем. Принимает запросы на списание с карты, имитирует задержку (1.5 сек) и в 90% случаев возвращает успех, в 10% — ошибку. БД отсутствует. |
